Documentação para desenvolvedores

O que este dashboard coleta e como

Referência técnica dos dados lidos do LinkedIn para alimentar cada componente do dashboard. A leitura é disparada pelo compartilhamento do certificado pelo beneficiário.

Gatilho de coleta

A coleta é iniciada quando o beneficiário compartilha o certificado POK no LinkedIn. Este evento é o D+0 — marco zero de toda a linha do tempo. A partir deste momento, a POK lê o perfil público completo e agenda leituras periódicas automáticas em D+30, D+90 e D+180. O histórico anterior ao compartilhamento é lido retroativamente a partir dos dados já presentes no perfil (Experience, Education, Certifications com suas datas originais).

Experience
  • → Título do cargo (title)
  • → Nome da empresa (company.name)
  • → Setor da empresa (company.industry)
  • → Data de início e fim (startDate / endDate)
  • → Emprego atual (flag isCurrent)
Usado em: cargo detectado na tabela · nº de cargos e empresas na curva · detecção de novo emprego e promoção
Skills
  • → Lista de habilidades declaradas (skills[].name)
  • → Quantidade total de skills no perfil
  • → Número de endorsements por skill (endorsementCount)
Usado em: nº de skills listadas na curva · evolução pré e pós compartilhamento
Licenses & Certifications
  • → Nome do certificado (name)
  • → Organização emissora (issuingOrganization)
  • → Data de emissão (issueDate)
  • → Quantidade total de certificados no perfil
Usado em: nº de certificados na curva · verificação se o certificado POK foi adicionado ao perfil
Education
  • → Instituição (schoolName)
  • → Grau de formação (degree)
  • → Área de estudo (fieldOfStudy)
  • → Período (startDate / endDate)
Usado em: filtro por título · segmentação futura de impacto por nível de formação
Headline & About
  • → Texto do headline (headline)
  • → Presença e comprimento do About (summary)
Usado em: detecção de atualização do perfil pós-compartilhamento (D+30, D+90)
Activity (publicações)
  • → Detecção da postagem do certificado no feed
  • → Data da publicação (publishedAt)
  • → Adição do certificado à seção Licenses do perfil
Usado em: funil (publicações detectadas · adicionado ao perfil) · ícones de LinkedIn na tabela
Lógica de detecção de impacto
Novo emprego

Comparar company.name do cargo atual no snapshot D+0 com o mesmo campo em snapshots posteriores. Se isCurrent = true e a empresa mudou → novo emprego detectado.

Promoção

Comparar title do cargo atual entre snapshots. Se a empresa permanece igual mas o título mudou para um nível superior → promoção detectada.

Dias até impacto

Diferença em dias entre a data do compartilhamento (sharedAt) e a data de início do novo cargo (startDate) detectado na leitura posterior.

Senioridade

Mapeamento do title para escala numérica (1–5): Estágio/Júnior → 1, Pleno → 2, Sênior → 3, Gerente/Especialista → 4, Diretor/VP/C-Level → 5.
